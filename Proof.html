HSBC — multi-year agentic AI with Google Cloud

Global universal bank · announced 17 Jun 2026 at Google Cloud Summit London
Built
A multi-year agreement giving HSBC priority access to Gemini models + the Gemini Enterprise Agent Platform, with HSBC + Google Cloud + Google DeepMind engineers co-developing tools across wealth, financial-crime risk and frontline decision support.
How it works
Financial-crime workflows screen the bank's transaction flow via agentic AI; an internal decision assistant cuts client meeting prep from hours to minutes for thousands of relationship managers.
Results
Commits to 200+ new AI use cases over 2 years; selected use cases each projected to deliver >$100M in revenue or efficiency; targets 2x faster intervention on financial-crime alerts across ~1B transactions monitored monthly.
The catch
Headline numbers are firm-stated projections, not audited outcomes; deepens HSBC's reliance on a single cloud + lab stack just as 'single-vendor AI risk' becomes a procurement theme.
Next
Roll-out across global markets through 2027. HSBC

Samsung Electronics — ChatGPT + Codex group-wide

World's largest electronics maker · disclosed by OpenAI 21 Jun 2026
Built
A firm-wide deployment of ChatGPT Enterprise + Codex to all Samsung Electronics employees in Korea and the global Device eXperience (DX) division — covers research, manufacturing, marketing, admin and software development.
How it works
ChatGPT runs under Samsung's enterprise data-protection / RBAC controls; Codex lets technical and non-technical teams ship internal tools, websites, automations — turning ideas into working software via prompts.
Results
OpenAI calls this one of its largest enterprise rollouts to date; Codex weekly active users in Korea up ~800% since 1 Feb.
The catch
Productivity gains haven't been disclosed; Samsung is also a major Anthropic customer (Samsung SDS), so this is multi-vendor by design — vendor concentration mitigated at the group, not the device-eXperience, level.
Next
Scaling beyond DX; broader Samsung Group rollout to follow. OpenAI

NAVER — Claude Code across all engineering

Korea's largest internet company · disclosed at Anthropic's Seoul opening, 17 Jun 2026
Built
A firm-wide rollout of Claude Code (Anthropic's terminal/IDE coding agent) to the entire NAVER engineering organisation — alongside Samsung SDS rolling Claude across Samsung Electronics and LG CNS across LG Group.
How it works
Developers use Claude as a peer that reads the repo, runs tasks in the terminal, edits files and opens pull requests — the same model Anthropic itself uses to write 80%+ of its own code.
Results
Headline-grabbing scope, not yet headline-grabbing metrics; results from comparable rollouts (Anthropic internal, Goldman Devin) suggest 3–8× more code committed per dev per day on repetitive work.
The catch
Korea is currently restricted from Anthropic's top Mythos/Fable tier under US export rules — NAVER is betting on the standard (Opus-class) line; capability ceiling depends on a policy decision in Washington.
Next
Anthropic-Korea MOST MoU on AI safety (19 Jun); 60 researchers across KAIST/Korea/Yonsei/POSTECH get free access. Anthropic

Morgan Stanley — opening the pipes to outside agents

First US bank to expose core platforms to external AI via MCP · announced June 2026
Built
MCP servers in front of ShareWorks and Equity Edge — the stock-plan administration platforms that hold $1.2 trillion in assets across ~3,400 corporate clients.
How it works
Anthropic's open Model Context Protocol (MCP) lets the client's own AI agent call MS data and workflows directly — vesting schedules, tax handling, tailored reporting — bypassing the human-facing UI.
Results
Live with a small early-access group; broader rollout planned across all ~3,400 administration clients by 2027.
The catch
No public ROI numbers yet — this is a strategic move ("be the platform agents call") rather than a measured win; also a new audit surface every regulator will want to inspect.
Next
Expect competitors (JPM, GS) to choose: open up via MCP or stay walled-garden. CNBC

JPMorgan Chase — agents that run for hours

From 230K-user LLM Suite to long-running autonomy · 2026
Built
A model-agnostic internal assistant ("LLM Suite") wired into the bank's own documents/data via secure APIs, now layered with longer-running agents in private banking and engineering.
How it works
Models from OpenAI and Anthropic run behind the firewall; per chief analytics officer Derek Waldron, new agents this year will "run for an hour or two" on a single goal instead of a few minutes.
Results
230,000+ users, 30–40% efficiency gains, contract review −40%; overnight market-screening for private bankers drove a ~20% gross-sales lift. Internal target: 450+ AI use cases live, 1,000 by year-end.
The catch
Hour-long autonomy is still a 2H 2026 plan, not a deployed metric; underwritten by an $18B tech budget few peers can match.
Next
"Long-running autonomous agents" across corporate workflows. CNBC

Commonwealth Bank — agentic fraud defence

Australia's largest bank · in production 2026
Built
An agentic system that spots emerging fraud patterns and auto-writes the detection rules to block them — built in-house in ~3 months.
How it works
Monitors 80M+ signals a day across transactions, cards, payments and digital channels, then drafts/updates fraud rules for human review.
Results
Fraud losses down >20% (H1 FY26 vs FY25); ~40,000 proactive customer alerts/day; the agent contributed to ~75% of card-fraud rules.
The catch
Part of a $1B/yr financial-crime program — runs alongside a separate ~$1B suspected loan-fraud probe; rules still need human review.
Next
Extending agentic detection to more channels. CommBank newsroom

FIS + Anthropic — Financial Crimes AI agent

Core-banking vendor + AI lab · announced May 2026, GA H2 2026
Built
An agent-first AML system: Anthropic's Claude reasoning embedded in FIS's banking data and regulatory infrastructure, co-built by Anthropic's forward-deployed engineers.
How it works
The agent assembles evidence across a bank's core systems, scores activity against money-laundering typologies, and surfaces the highest-risk cases — every decision traceable, client data stays inside FIS infrastructure.
Results
Designed to cut AML investigations from hours to minutes, reduce false positives and lift SAR narrative quality.
The catch
Still early — BMO and Amalgamated Bank are in development, GA only planned for H2 2026; results are design targets, not audited outcomes.
Next
Roadmap spans credit decisioning, deposit retention, onboarding and fraud. FIS / Business Wire

Klarna — the cautionary tale

Fintech · AI-only support, then a partial reversal
Built
An OpenAI-built assistant that replaced ~700 customer-service roles, handling chat in 35+ languages.
Results
Peaked at ~75% of chats (~2.3M conversations), 82% faster responses, 25% fewer repeat contacts.
The catch
Quality slipped. The CEO admitted Klarna "over-rotated"; by 2025 it began rehiring humans for premium and complex cases.
Next
A blended model — AI for volume, humans for premium. The lesson: measure quality, not just cost. Reuters/Bloomberg via Entrepreneur
